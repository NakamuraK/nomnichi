Gitを使用したnomnichi開発

このページでは，kawasemi上のホームディレクトリでnomnichiの更新を行う方法を例に，Gitを利用した開発方法について説明する．
なお，このページで扱うnomnichiはRailsで開発されているものを指す．
また，nomnichiは現在GitHubで管理されている．

Gitを使用してnomnichiを更新

  1. ローカルリポジトリを作成
    $ git clone git@github.com:nomlab/nomnichi-rails.git /home/home_directory_name
  2. ファイルを編集
  3. 編集内容を確認
    1. ローカルリポジトリの状態を確認
      $ git status
    2. ファイルの変更点を確認
      $ git diff
    3. Web上で確認
      1. 必要なファイルを稼働中のものからコピー
        $ cp /home/rails/nomnichi/production/config/database.yml config
$ cp /home/rails/nomnichi/production/db/development.sqlite3 db
$ cp -r /home/rails/nomnichi/production/db/large_objects db
      2. データベーステーブルの作成
        $ rake db:migrate
      3. サーバ立ち上げ
        $ rails s -p 12345(お好きなポート番号)
      4. Web上で確認
        以下のurlにアクセスして確認
        http://kawasemi.swlab.cs.okayama-u.ac.jp:12345
      5. サーバ停止
        Ctrl + c
  4. 編集したファイルをadd&commit
    1. 編集したファイルをindexにステージ
      $ git add file_path
    2. indexにある変更点をコミット
      $ git commit
    3. コミットメッセージを入力し，保存して終了．(Ctrl + x -> yes -> Enter)
  5. コミットを確認
    1. コミットログを確認
      $ git log
    2. 編集内容を確認
      $ git show
  6. リモートリポジトリに変更点をpush
    リモートリポジトリのmasterブランチに変更点を送信
    $ git push origin master
