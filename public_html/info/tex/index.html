TeX による論文執筆のすすめ

作成者：乃村 能成
作成日：2008/1/30
更新日：2009/10/22 (牛尾 裕)

1. はじめに

卒業論文の原稿を TeX で書くことをすすめる．
大抵の学会では，論文投稿用の TeX スタイルファイルを提供している．
卒論の内容を発展させて学会発表をすることを考えると，
TeXによる原稿作成をしておくことが省力化に役立つからである．

ただし，TeX で卒論を書いたとしても，全く我流で書いてしまうと
不要な苦労が増えることになる．

そこで，本文書では，TeX で論文等を記述する際に あらかじめ注意すべき事項について解説する．

なお，本文書は，TeX の文法や
文章の書き方の解説ではない．
それらは，別途，教本等を参照のこと．

また，WindowsでTeX環境を構築する手順は以下の通りである．
(1) 以下のリンクからTeXインストーラの最新バージョンをダウンロードする．
TeXインストーラのダウンロードページ
(2) インストーラを実行する．選択肢は変更しなくても良い.
　　<注意>
　　　(a)インストーラを起動後，TeXのダウンロード先URLを幾つか聞かれます．
　　　　　このとき，デフォルトのURLからは，ダウンロードできません．
　　　　　一覧の下から2番目のURLを指定すると，幾分かダウンロードがしやすいです．
　　　　　これでもダメな場合は，他のURLをしつこく試してください．
(3) 以下のページから，適当なエディタをダウンロードする．
TeXエディタのダウンロードページ
(4)エディタの設定を変更する．

2. ファイル構成

標準的なディレクトリ構成を守っておくと作業がしやすくなる．

(1) b4-thesis.tex -- 論文本体
(2) b4-thesis.ppt -- 発表用のスライド
(3) fig/ -- グラフや
(4) 　all-figs.ppt -- 論文中で使用する図の原版
(5) 　fig01.eps -- all-figs.ppt から抽出した 図1
(6) 　fig02.eps -- 同 図2 以下必要分の eps ファイル

このとき，以下の事項を注意するとよい．

(1) 不必要に TeX ファイルを分割しない．

章毎にTeXファイルを分割して，メインのファイルから
インクルードしている記述方法が見受けられるが，
これは主に執筆者が複数分担している場合や，100頁を越える
博士論文や書籍などで用いる手法である．
個人で記述する数十ページの論文には，すすめない．

(2) 図表はサブディレクトリに入れておく

図表を過去の複数の自筆論文から再利用する場合がある．
その場合は，fig-conf-okinawa2007/ fig-conf-sapporo2006/
のように複数に分けて配置すると，再利用がしやすい．

(2) 図版のファイル名は，機械的でもよい

本来 TeX の精神に沿えば，図版のファイル名は，
fig01.eps のような出現順を意識した方法ではなく，
cpu-usage-graph.eps のように，内容を表す名前にすべきである．
しかしながら，この方法は，後から見た場合に，
意外と混乱しやすい．

図版枚数や個人の感覚にも依るが，図が10枚を越えなければ，
経験上，割切って fig01 とした方が混乱が少なく，組版後の論文
と原稿との対応を付けやすい．

(3) 日本語のファイル名は避ける

TeX の可搬性が損われる可能性がある．特に，tar や zip に
アーカイブした場合に file 名の漢字コードが問題になる．
学界にファイルのコピーを送付する場合の問題にもなる．

3. 図(EPS)の取り込み

(1) PPT→EPS 変換は，WMF を経由するとよい

PPT から図を EPS に変換して取り込むのが一般的である．
その際，WMF (Windows Meta Format) を経由して変換すると
失敗が少なく，綺麗に変換できる．GIF や PNG などから
変換して拡大した場合は，印刷時の品質が著しく落ちるので，
気を付けること．

「WMF2EPS」 というツールを勧める．

(2) 取り込みの書式をマクロで統一する

論文は 2カラムであるため，図表取り込みの幅を
マクロで一括変更できるようにしておくと効率がよい．
また，ディレクトリを一ヶ所で設定しておくと，
図の再利用がしやすい．以下は，カラム幅一杯，中央に
図を挿入するマクロの例である．


\def\figdir{fig}
\def\insertfigure#1#2#3#4{
\begin{figure}[ht]
\begin{center}
\vspace{1em}
\includegraphics[width=\columnwidth]{\figdir/#2}
\caption{#3\label{#1}}
\ecaption{#4}
\vspace{1em}
\end{center}
\end{figure}}

\columnwidth 変更することで，一括して図の幅を調整できる．
利用には，

\def\figdir{my-fig-dir}

として図のディレクトリ指定を文書の最初で指定しておき，

\insertfigure{fig:fig2}{fig2.eps}{表題…}{english}

として各図を挿入していく．fig:fig2 はラベル．
以下，挿入ファイル名，和文キャプション，
英文キャプションと続く．

経験では，カラム幅一杯に図を挿入するマクロと，
用紙印刷範囲一杯に図を挿入するマクロがあれば，
ほとんど用が足りる．\ecaption が不要であれば，
コメントアウトするなり，空にマクロ
定義しておけばよい．

(3) 図版の大きさを考慮する

卒論は1カラムに対して，論文は，2カラムなので，
あらかじめ 2カラムで図版の大きさを考慮しておく．
つまり，半分の幅に入れても読みにくくならないか
考慮する．

4. ラベルと引用

(1) 図と表の参照は専用定義する

\def\tabref#1{
表\ref{#1}
}
\def\figref#1{
図\ref{#1}
}

(2) ラベルは，プレフィクスを付ける

図であるか，表であるかを区別するプレフィクスを付けておく
とよい．例えば，

fig:figurename
tab:tablename

などである．また，fig:dps99:figure のように，
原稿毎にプレフィクスを用意しておくと，
過去の論文から部分的な引用がしやすい．

(3) 引用の書式を統一する

論文誌の書式と同じになるようにしておくと後々楽である．

(4) ラベルに日本語を書かない

\label{はじめに} などとしない．TeXの互換性から問題を起こす可能性がある．
たとえば，IEEE 英文論文は，TeX 原稿の提出を求められる場合がある．
(あちらでコンパイルして互換性をチェックされる) その際に問題になる．

5. 書式の統一

(1) 適当な位置での改行をこころがける

TeX や HTML など，構造を持った文書では，
インデントを付けたり適当な位置での改行が，
後々の見やすさにつながるので日頃から気をつける．
その際，改行位置は，文章の文節にしておくこと．
検索や一括の置換の便がよい．

(2) 句読点，全角半角の区別を統一する

論文誌では，句読点を 「，．」に指定しているので，日頃から
合わせておくと混乱が少ない．さもなくが，
日頃の資料からの引用で論文を構成した場合に，
句読点置換を忘れないこと．

(3) 全角半角の使い分け

アルファベットは，カーニングとハイフネーションのために，
半角で書くこと．それ以外は，あまり拘り過ぎなくてもよい．
ちなみに，（）など，日本語文章中は全角だが，数式中では
半角にすべきものなどがある．数字も文章中の１桁の数字は，
全角としている．

(4) twocolumn での組版を考慮に入れる

2カラムで組版しても文書が崩れないように書くとよい．
そのためには，無理矢理改頁を入れて図版の位置を
調節するなどしない．

6. その他

(1) カナ表記と送り仮名を統一する

漢字で記述するもの，カタカナ表記の揺れなどは，
論文誌を読んで，それらの用語に合わせておくとよい．

(2) 機種依存文字を使わない

○に1や，1文字で(株)となるなどの，
いわゆる機種依存文字，あるいは半角カナは使用しない．
