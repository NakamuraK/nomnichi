■ 利用するクラスファイル

卒論/修論作成用のクラスファイルとして年度ごとに graduate(その年度).cls が用意されます。
今年度の場合、graduate2008.cls が用意されていますので、これを利用してください。
具体的には TeX のソースファイルの先頭に次のように記述します。

\documentclass{graduate2008}
\usepackage{makeidx}

makeidx は索引を付ける際に必要となるパッケージです。
卒論/修論の索引は必須ですから、これも使用するように設定します。

注意:必ずその年度のクラスファイルを 使ってください。
卒論のスタイルファイルは、年度により索引、マージンなどが異なります。

管理者へ: graduate*.{cls,sty} を直接変更しないように。
昔の人の卒論の見た目が変わったり、最悪コンパイルできなくなります。
変更がなくても、毎年 graduate[0-9]*.{cls,sty} を作りましょう。

■ 図の作成

現在、論文中に入れる図を作成する一般的な手法には、以下の2つがあります。

  1. Unix 上でTgifを利用する。
  2. Windows 上で PowerPoint を利用する。
どちらの方法でも日本語を使うことはできます。 各自、好みの方法で図を作成してください。 Tgif と PowerPoint それぞれの利点は以下の通り。
Tgif の利点
  1. UNIX マシンだけで、TeX文書と図共に作成することができるため効率が良い。
  2. 比較的まともな PostScript を出力してくれるので、プレビューや印刷時のトラブルがほとんど無い。
  3. 作成される EPS ファイルのファイルサイズが、PowerPoint で作成するものよりずっと小さい。(1/10 以下)
PowerPoint の利点
  1. TeX 用に作成した図をそのままプレゼンテーションに使用できる。
  2. 使い方を覚えるまでに時間がかからない。

PowerPoint で作成する際の豆知識

PowerPoint で作成した EPS ファイルには、無駄な余白がついてしまい、TeX でのレイアウトに苦労します。 このアナウンスにある /usr/local/bin/psfixbb (Perlスクリプトなのでどの計算機上でも動きます)コマンドを利用すると、この『無駄な空白』を取り除くことができます。

また、PowerPoint を利用して EPS ファイルをつくる際には、[ファイル]→[ページ設定...]のダイアログで『スライドのサイズ指定』を A4 にしておきましょう(通常は「画面に合わせる」になっています)。 縮尺の計算が楽になります。

TeX ファイルで EPS ファイルを取り込むには、\includegraphics コマンドを利用します。 この際に、``clip'' オプションを使用すると枠外にでる『謎の空白』が出なくなるため、とても幸せになれます。

【例】 

\begin{figure}[htbp]
 \begin{center}
      \includegraphics[clip]{foobar.eps}
  \caption{Foo Bar}
  \label{fig:foobar}
 \end{center}
\end{figure}

■ 画面のダンプイメージを入れたいときには

プログラムの実行画面などを論文にいれたい場合に、画像→EPSへの変換で一般的に行われる

  1. 画面のダンプを取る。
  2. ペイントツール等に張り付ける。
  3. 印刷助の出力先を EPS ファイルとする。
というやり方は、できあがるファイルサイズが大きくなってしまい、論文のファイルサイズが大きくなる＆印刷に時間がかかってしまう、という問題があります。

以下の方法のいずれかを用いると、多くの場合ファイルサイズを小さくすることができ、結果、印刷時間を短縮することができます。(たまに逆に大きくなってしまう場合もあります)

  1. /usr/local/bin/convert コマンドを利用する。
  2. /usr/local/X11/bin/xv-3.10a コマンドを利用する。
  3. jpeg2ps ツールを利用する。
    1. ダンプイメージを JPEG ファイルとして保存する。
    2. jpeg2psを利用して EPS ファイルに変換する。(crest にインストール済み。ソースファイルはこちら) 
      jpeg2ps V1.8: convert JPEG files to PostScript Level 2.

■ 図の張り付け

図を張り付ける時は graphicx パッケージを利用します。 

\documentclass{graduate2000}
\usepackage{makeidx}
\usepackage[dvips]{graphicx}

そして図(ここでは hoge.eps とする)を載せたい箇所で、 次のように書きます。

このような情報ベースシステムの概念図を図 \ref{fig:kouseizu} に示す。

\begin{figure}[hbt]
  \begin{center}
    \includegraphics[clip,scale=.5]{hoge.eps}

    \caption{情報ベースシステムの概念図}
  \end{center}
  \label{fig:kouseizu}
\end{figure}

scale は図の大きさを調整するためのスイッチです。
この例では hoge.eps を0.5倍(半分)にして張り付けています。

注意:標準の設定では、 図や表のキャプションに":"が入ります。
この":"を出したくない場合は、\documentclass{graduate2000}
のところで、no-colon オプションを指定してください。つまり、
\documentclass[no-colon]{graduate2000} としてください。

■論文の体裁「はじめに」と「おわりに」

論文本文を「はじめに」という章ではじめたら、 「おわりに」という章で対応をとりましょう。 そのためには、次の二つのキーワード\preface{...} \conclusion{...}を使います。 \preface{...}は論文本文の直前、 \conclusion{...}は論文本文の最後と謝辞の間で使用します。

なおこの場合「はじめに」と「おわりに」には章番号は振られません。

\preface{はじめに}

  近年、爆発的な広がりを見せたインターネットでは...
\chapter{マルチキャスト}
  ... %論文本文
\conclusion{おわりに}
  本論文では...について述べた。今後の課題として次のものが残っている。
\acknowledgement

■ 付録の書き方

付録は\appendixの後に記述します。

\appendix
\chapter{…の全ソース}

\section{foo.c}

\small
\begin{verbatim}
#include<stdio.h>
main(){
  printf("hello world.");
}
\end{verbatim}

■ 索引の作り方

計算機研では、卒論・修論に索引を付けることを義務づけています。
索引を付けるには次のようにします。

  1. makeidxパッケージを利用するように設定する(済)。

    \documentclass{graduate2000}

\usepackage{makeidx}

  2. 索引を出力したい位置(通常は\end{document}の直前)に 次のように記述する。

    \insertindex
\printindex

  3. 索引の対象とする用語にインデックスを付ける。

    • 普通に索引を付ける

      語句\index{語句のよみ@語句} という具合に追加します。

      この章では、並行型プログラムのテストに関して調査した
結果について報告する\index{てすと@テスト}。

    • ある索引にサブ項目を入れる場合

      「単体テスト」 「受入テスト」 を 「テスト」 のサブ項目と して入れたい場合。 

      上流工程に対応した、単体テスト\index{てすと@テスト!たんたい---@単体---}
      受入テスト \index{てすと@テスト!うけいれ---@受入---}を行う。

    注意

    • \index{...} は、1行に全部書く。途中で改行しない。
    • 語句\index{よみ@語句} と書くときに、語句と \index{..} の間に
      スペースや改行を入れないで密着させる。

  4. platex, jmakeindex, platex を順に実行する。

    % platex     filename.tex  (filename.idx が出来る)
% jmakeindex filename.idx  (filename.idx から filename.ind を生成)
% platex     filename.tex  (filename.ins を使う)
